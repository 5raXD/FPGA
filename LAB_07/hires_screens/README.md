# High-resolution welcome / game-over screens

Drop-in upgrades for the blocky 100×75 IDLE and GAME_OVER screens. The gameplay
grid stays 100×75 — only the two full-screen images get sharper. Bitmaps are
generated by `~/.claude/skills/fpga-pixel-art/hires_screens.py` into
`imgs_pixils/pics_hd/{low,med,high}/`; browse each tier's `gallery.html`.

Board: **Basys3 XC7A35T** — 20,800 LUTs, 50 × 36 Kbit = **1,800 Kbit BRAM**.
Only one welcome + one game-over screen are loaded at once, so "strain" below
is for **two** screens.

## Pick a tier by strain

| tier | resolution | per screen | two screens | what you change |
|------|-----------|-----------|-------------|-----------------|
| **low**  | 200×150, 1-bit | ~30 Kbit LUT-ROM | a few % LUTs, **0 BRAM** | just edit `GridMapper` (no new module) |
| **med**  | 400×300, 1-bit | 120 Kbit BRAM | ~8 BRAM blocks (**~16%**) | add `Screen_ROM` |
| **high** | 400×300, 16-colour | 480 Kbit BRAM + palette | ~27 BRAM blocks (**~54%**) | add `Screen_ROM_color` |

**Recommendation:** start with **low** (zero BRAM, same architecture as today,
4× the detail — text and skull already look far better). Go **med** if you want
razor-sharp edges, **high** only if you want full-colour art and don't mind
spending half the BRAM.

Coordinate scaling from the 800×600 VGA counters is a plain shift:

| resolution | x | y |
|-----------|---|---|
| 200×150 | `XCoord>>2` | `YCoord>>2` |
| 400×300 | `XCoord>>1` | `YCoord>>1` |
| 800×600 | `XCoord`    | `YCoord`    |

---

## Tier `low` — no new module (safest)

Copy `pics_hd/low/<welcome>.mem` and `<gameover>.mem` next to the Verilog, then
in `GridMapper.v` widen the arrays and change the shift from `>>3` to `>>2`:

```verilog
localparam SW = 200, SH = 150;               // screen bitmap size
reg [SW-1:0] welcome [0:SH-1];
reg [SW-1:0] skull   [0:SH-1];
initial $readmemb("wc_snake_text.mem", welcome);
initial $readmemb("go_skull_youdied.mem", skull);

wire [7:0] sx = XCoord >> 2;                  // 0..199
wire [7:0] sy = YCoord >> 2;                  // 0..149
wire on_welcome = welcome[sy][SW-1-sx];
wire on_skull   = skull[sy][SW-1-sx];
```

Colours stay the FG/BG constants you already have (each `pics_hd` preview shows
the intended pair). This is the same combinational ROM you use now, just bigger.

## Tier `med` — `Screen_ROM` (sharp mono, ~16% BRAM)

Instantiate one ROM per screen and mux by state. Feed the shifted coordinates
directly; the module registers the row read so Vivado uses BRAM.

```verilog
wire [8:0] sx = XCoord >> 1;                  // 0..399
wire [8:0] sy = YCoord >> 1;                  // 0..299
wire on_welcome, on_skull;

Screen_ROM #(.W(400), .H(300), .XW(9), .YW(9),
             .MEMFILE("wc_snake_text.mem"))
    welcome_rom (.clk(clk), .x(sx), .y(sy), .pixel(on_welcome));

Screen_ROM #(.W(400), .H(300), .XW(9), .YW(9),
             .MEMFILE("go_skull_youdied.mem"))
    skull_rom  (.clk(clk), .x(sx), .y(sy), .pixel(on_skull));

// then, as today:
// IDLE:      block_color = on_welcome ? WELCOME_FG : WELCOME_BG;
// GAME_OVER: block_color = on_skull   ? GAMEOVER_FG : GAMEOVER_BG;
```

## Tier `high` — `Screen_ROM_color` (full colour, ~54% BRAM)

Compute a linear address and read a 12-bit colour straight out. The palette is
baked into the `.mem`, so no FG/BG constants needed.

```verilog
wire [8:0]  sx   = XCoord >> 1;
wire [8:0]  sy   = YCoord >> 1;
wire [16:0] addr = sy * 400 + sx;             // one DSP/mult, or precompute
wire [11:0] welcome_col, skull_col;

Screen_ROM_color #(.W(400), .H(300), .AW(17),
        .IMG("wc_snake_art.mem"), .PAL("wc_snake_art_pal.mem"))
    welcome_rom (.clk(clk), .addr(addr), .color(welcome_col));

Screen_ROM_color #(.W(400), .H(300), .AW(17),
        .IMG("go_rip.mem"), .PAL("go_rip_pal.mem"))
    skull_rom  (.clk(clk), .addr(addr), .color(skull_col));

// IDLE:      block_color = welcome_col;
// GAME_OVER: block_color = skull_col;
// PLAY:      (unchanged snake/food/grid colouring)
```

## Timing note

Both ROMs have **1 clock of latency**. Your `VGA_Interface` already registers
`pixel_color` and runs 2 clocks per pixel (`pix_en`), and the screen `y` is
constant across a scan-line, so the only effect is the first pixel of each line
— inside the blanking margin, invisible. If you ever see a 1-pixel left-edge
artifact, register `sx`/`addr` one extra cycle to match.

## Files

- `Screen_ROM.v` — 1-bit BRAM image ROM (tiers med, and reusable at 800×600).
- `Screen_ROM_color.v` — 16-colour BRAM image ROM + palette (tier high).
- Bitmaps: `imgs_pixils/pics_hd/<tier>/`. Regenerate with
  `python3 ~/.claude/skills/fpga-pixel-art/hires_screens.py`.
