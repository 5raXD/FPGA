Job

On every tick: move the head in the current direction, follow with the body, grow on food, drop new food, check collisions, update the score, and run the game state machine.

Suggested ports

module Snake_Logic #(parameter GRID_X=40, GRID_Y=30, MAX_LEN=64)(
    input  wire        clk, rstn,
    input  wire        tick,          // from Game Tick
    input  wire        start,         // debounced start/restart button
    input  wire [1:0]  dir_next,      // from Direction Control
    input  wire [5:0]  cand_col,      // from LFSR
    input  wire [5:0]  cand_row,
    // to Renderer (queried for the cell currently scanned):
    input  wire [5:0]  q_col, q_row,
    output wire        cell_is_snake,
    output wire        cell_is_head,
    output reg  [5:0]  food_col, food_row,
    output reg  [1:0]  dir,           // committed direction (to Direction Control)
    output reg  [11:0] score,         // to Seg7 Display
    output reg         game_over      // to Renderer + FSM
    );

What happens on each tick

dir <= dir_next; — commit the buffered turn (one turn per move).
Compute the new head cell from dir (remember: Up = row−1).
Wall check: new head outside 0..GRID_X-1 / 0..GRID_Y-1 → game_over.
Self check: new head lands on an occupied body cell → game_over.
Food check: new head == food → score++, grow (don't drop the tail), respawn food via the LFSR (re-roll if it hits the snake).
Otherwise move normally: add the new head, remove the tail.

Pick a body representation

This choice is the design. Both are explained in depth on the main page (§4):

ApproachIdeaBest when
A — segment arraystore up to MAX_LEN (x,y) pairs; shift them forward each ticksmall snake; easiest to get working & simulate
B — grid + countdownone value per cell = ticks-until-empty; tail clears itselflong snake; O(1) render & collision

→ full explanation with diagrams on the main page

With Approach B, cell_is_snake/cell_is_head for the Renderer are just memory reads at (q_col,q_row) — which is why it renders so cleanly at 50 MHz.

Game state machine

Reset everything on restart. Returning to IDLE must re-init snake body, length, dir, score, and food — or your next game starts with the previous corpse.
