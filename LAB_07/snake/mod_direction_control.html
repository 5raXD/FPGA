Job

Translate a keypad scancode into a 2-bit direction, reject a 180° reversal, and buffer the choice until the next game tick.

Suggested ports

module Direction_Control(
    input  wire       clk,
    input  wire       rstn,
    input  wire [7:0] scancode,     // from Ps2_Interface
    input  wire       keyPressed,   // from Ps2_Interface (one-shot)
    input  wire       tick,         // from Game Tick
    input  wire [1:0] dir,          // current committed direction (from Snake_Logic)
    output reg  [1:0] dir_next      // the legal requested direction
    );

Key mapping (numeric keypad, from Lab 5)

KeyDirectionMake-code(dx,dy)
4Left0x6B(−1, 0)
6Right0x74(+1, 0)
8Up0x75(0, −1)
5Down0x73(0, +1)

Implementation idea

localparam L=2'b00, R=2'b01, U=2'b10, D=2'b11;

always @(posedge clk) begin
    if (!rstn) dir_next <= R;
    else if (keyPressed) case (scancode)
        8'h6B: if (dir != R) dir_next <= L;   // can't reverse Right->Left
        8'h74: if (dir != L) dir_next <= R;
        8'h75: if (dir != D) dir_next <= U;
        8'h73: if (dir != U) dir_next <= D;
    endcase
end
Buffer, then commit on the tick. Latch into dir_next here; Snake_Logic copies it into dir once per tick. Otherwise two fast presses between ticks let the snake reverse via two 90° turns. (requirement 3.b)
Y axis is inverted on screen: Up = row−1, Down = row+1 (row 0 is the top).
