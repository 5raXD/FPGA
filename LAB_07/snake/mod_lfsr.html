LFSR — Pseudo-Random Food

Tel Aviv University · FPGA Lab · Experiment 7 — Snake · Module Guide

new — short, you write it ~3 lines of Verilog. Nothing to import.

Job

Give the snake "random" food positions. An FPGA has no rand(), so use a Linear-Feedback Shift Register: a shift register whose input bit is the XOR of a few tap bits. It free-runs and cycles through values in a scrambled order.

Never let it be all-zeros. 0 XOR 0 = 0, so an all-zero register shifts in zeros forever and freezes. Seed it with a non-zero constant (e.g. 16'hACE1) — that's why the period is 2¹⁶−1 = 65 535, not 65 536.

Implementation idea

reg [15:0] lfsr = 16'hACE1;             // any non-zero seed
always @(posedge clk)
    lfsr <= {lfsr[14:0], lfsr[15]^lfsr[13]^lfsr[12]^lfsr[10]};

// squeeze into the grid when you need a new food cell:
wire [5:0] cand_col = lfsr[5:0]  % GRID_X;
wire [5:0] cand_row = lfsr[11:6] % GRID_Y;
Don't spawn on the snake. After computing a candidate cell, ask Snake_Logic if it's occupied; if so, just take the next LFSR value (it changes every clock) until you find an empty cell, then latch it as the new food.
Let it free-run. If you only clock the LFSR when food is eaten, the position depends solely on how many foods you've eaten — predictable. Free-running ties it to real time, which feels random.

Connects to

cand_col/cand_row → Snake_Logic, which validates and stores food_col/food_row (then read by the Renderer).
