What to do

The 4-digit 7-segment driver is one of your oldest modules. Copy whichever copy you trust into the Lab-7 project: 

// Any of these is the same module — pick the latest:
LAB_05/Seg_7_Display.v
LAB_04/Seg_7_Display.v
LAB_03/Seg_7_Display.v

It already multiplexes the four digits fast enough that the eye sees a steady number.

Ports (already defined)

module Seg_7_Display(
    input  wire [15:0] x,         // the number to show (4 hex nibbles)
    input  wire        clk,       // 100 MHz
    input  wire        clr,       // clear/reset
    output reg  [6:0]  a_to_g,    // segment cathodes (active low)
    output reg  [3:0]  an,        // 4 digit anodes  (active low)
    output wire        dp         // decimal point
    );

How it plugs into the game

Seg_7_Display u_seg(
    .x({4'b0, score[11:0]}),   // pad the score up to 16 bits
    .clk(clk), .clr(~rstn),
    .a_to_g(seg), .an(an), .dp(dp));

Decimal vs. hex

x is shown as hex nibbles. If you want a human decimal score, convert score to BCD first (double-dabble) and feed the BCD digits in. Hex is perfectly acceptable for the lab — just state your choice in the report.

The score itself is owned by Snake_Logic — it increments by 1 on every ate_food. This module only renders whatever number you hand it.
