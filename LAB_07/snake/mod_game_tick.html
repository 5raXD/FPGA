Job

Produce a one-cycle pulse (tick) every ~150 ms. The whole game advances only when tick is high — that's what makes the snake move at a human speed instead of millions of cells per second.

Implementation idea

module Game_Tick #(parameter TICK_MAX = 15_000_000)(  // 100MHz*0.15s
    input  wire clk,
    input  wire rstn,
    output reg  tick
);
    reg [23:0] cnt;
    always @(posedge clk) begin
        if (!rstn)                 begin cnt <= 0; tick <= 0; end
        else if (cnt==TICK_MAX-1)  begin cnt <= 0; tick <= 1; end
        else                       begin cnt <= cnt+1; tick <= 0; end
    end
endmodule
Use a clock-enable, not a real slow clock. Stay in the 100 MHz domain and gate updates with if (tick) — same idiom Lab 6 used for the 50 MHz pixel enable. Generating a true 6 Hz posedge clock creates an extra clock domain and wastes a clock buffer.
Optional difficulty curve: shrink TICK_MAX as the score grows so the snake speeds up — make it a port driven by score.

Connects to

tick → Snake_Logic (move the snake) and Direction Control (commit the turn).
